## Knowledge Graph (kg)

The Knowledge Graph module exposes Harness's unified data model — the schema
types, queryable types, relationships, and data source connections that underpin
cross-module analytics and HQL queries.

### Domain Model

The schema is a typed graph. Every entity in Harness (project, pipeline,
service, deployment, etc.) maps to a schema type with a kind: ENTITY, EVENT,
METRIC, VIEW, RELATIONSHIP, or CONFIG. Types have fields and relate to other
types via directional relationship edges.

Queryable types are the subset of schema types reachable via HQL. Use
`list kg:queryable_type` to see what you can query before writing HQL.

Data source connections (`list kg:connection`) show which backends the query
engine is actively pulling from. A connection with `valid: false` means queries
against that type may return stale or empty results.

### HQL

Harness Query Language (HQL) is a graph query language modelled after ANTLR4
grammar. Basic form:

  find entity "platform:project" | select { * } | limit 10

Use `execute hql:validate` to check a query before running it, `execute
hql:explain` to inspect the execution plan, and `execute hql:run` to execute.
Fetch the full grammar with `execute hql:grammar`.
